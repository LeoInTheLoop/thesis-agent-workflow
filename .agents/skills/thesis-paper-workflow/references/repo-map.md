# Repository Map

Use this reference to find the active source of truth before changing files.

## Active Entry Points

- Workflow entry: `AGENTS.md`
- Skill entry: `.agents/skills/thesis-paper-workflow/SKILL.md`
- Thesis source: `thesis.tex`
- Main theme / RQ boundary: `paper_theme.md`
- Writing rules: `requirements/paper_guidelines.md`
- Terminology: `requirements/terminology/field_terms.md`
- Citation plan: `citation_ideas.csv`
- Bibliography: `tex/biblio.bib`
- Source PDFs: `ref/<BibTeX key>.pdf`
- Final-review checklist: `requirements/final_checklist.md`
- Final-submission numeric audit: `requirements/numbers_consistency.md` (create when first needed)
- Reflection draft: `requirements/reflection.md` (create when first needed)
- Modification history: `requirements/modification_summary.md`
- Open tasks: `todo.md`

> If your institution requires an official instructions PDF, store its path in `AGENTS.md` Section 0 and treat it as authoritative for formatting requirements.

## Historical Evidence Archives

Treat these patterns as evidence, not active todo lists, unless the user or `requirements/final_checklist.md` explicitly reactivates an item:

- `requirements/feedback_v1/`, `requirements/feedback_v2/`, … (older supervisor/reviewer rounds)
- `requirements/review_round_*/`
- `ppt/final/` or any archived defense-deck folders
- `reviewOthers/` or any peer-opposition evidence folders
- Anything explicitly named `archive/`, `old/`, or marked `Status: superseded`

## Paper Modification Boundary

- Edit paper content in `thesis.tex` only.
- If you use Overleaf or another remote editor, sync via the project's script (e.g., `python sync_overleaf.py from-overleaf`) after each editing pass. Do not edit two copies in parallel.
- Do not edit generated/split chapter files as the primary source unless the user explicitly asks for that direction.

## Status Updates

- Use `requirements/final_checklist.md` for final-submission status.
- Use `todo.md` for broader remaining tasks outside the final-final checklist.
- Use `requirements/modification_summary.md` for the round-by-round history.

## Canonical Folder Tree

This is the canonical layout. Whenever a new file appears (created, dropped in by the user, or extracted), match it against this tree **before** acting on it. If the file is not where this tree says it belongs, move it first, then continue.

## New-File Routing Rule (Mandatory)

When the agent sees a new file — one created in this session, dropped in by the user, or generated by a tool — run this triage **before** reading or editing it as content:

1. **Identify file type** by extension and apparent role.
2. **Match against the table below** to determine the correct location.
3. **If the file is already in the correct location**, proceed.
4. **If the file is in the wrong location**, stop and ask the user before moving:
   > "I found `<path>`. By the folder tree it belongs in `<expected path>` — should I move it?"
5. **If the file type is not in the table**, add a new row (and a `% Reason`) before placing the file, so future agents inherit the decision.

### File-Type Routing Table

| File type / pattern | Goes in | Notes |
|---------------------|---------|-------|
| `*.tex` (main thesis) | repo root as `thesis.tex` | Single source of truth. |
| `*.tex` (chapter splits) | `tex/` | Treat as generated, not primary, unless the user explicitly routes editing here. |
| `*.bib` | `tex/biblio.bib` | One bibliography file. Merge, do not multiply. |
| Source PDFs (cited papers) | `ref/<BibTeX key>.pdf` | Filename must equal the BibTeX key. PDFs are gitignored. |
| Extracted PDF text (OCR/pdfplumber) | `/tmp/<BibTeX key>.ocr.{pdf,txt}` | Transient evidence. Not committed. |
| Official institution instruction PDF | repo root, path recorded in `AGENTS.md` §0 | One canonical file. |
| Supervisor / reviewer feedback (current round) | `requirements/feedbackv<N>.md` | Live round in `requirements/` root. |
| Supervisor / reviewer feedback (older rounds) | `requirements/feedback_v*/` or `requirements/review_round_*/` | Evidence archive — do not resurrect tasks. |
| Writing rules / style guides | `requirements/paper_guidelines.md` | One file; merge into it instead of creating siblings. |
| Final-submission checklist | `requirements/final_checklist.md` | One file. |
| Numeric audit | `requirements/numbers_consistency.md` | Create on first numeric audit. |
| Reflection draft | `requirements/reflection.md` | Single file; first-person voice. |
| Terminology / glossary | `requirements/terminology/field_terms.md` | One canonical term list. |
| Citation planning CSV | repo root `citation_ideas.csv` | Single CSV; one row per BibTeX key. |
| Task / TODO list | repo root `todo.md` | One live task list. |
| Modification history | `requirements/modification_summary.md` | Append-only. |
| Defense slides / PPT (current) | `ppt/` | If `ppt/final/` exists, treat older subfolders as archive. |
| Defense slides / PPT (archive) | `ppt/final/` or `ppt/archive_*/` | Evidence; do not resurrect. |
| Peer-opposition evidence | `reviewOthers/` | Evidence archive. |
| Python tooling | `.venv/` | Local virtual environment, gitignored. |
| Workflow / agent rules | `AGENTS.md` (thin) + `.agents/skills/thesis-paper-workflow/` | Detailed rules belong in skill references, not at repo root. |

> **Why this rule exists**: New files tend to land at the repo root or in `/tmp/` by default; left uncorrected, the project accumulates duplicate-truth files (`thesis_v2.tex`, `citations_new.csv`, `feedback2.md` at the root) that silently diverge from the canonical ones. The routing rule forces a placement decision *before* the file is treated as content, so the canonical tree stays single-source.
